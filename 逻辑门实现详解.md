# 逻辑门实现详解

## SVG图形创建和定义
- **定义和存储**
  - 定义全局componentSvgs 对象，用于存储所有逻辑门的SVG模板
    - 使用 viewBox="0 0 200 100" 的统一坐标系统
    - 定义特定的css类标识系统，anchor为通用锚点类，用于事件绑定，data-anchor-type为数据属性，用于连线验证，input-anchor / output-anchor用来区分输入输出类型
- **与逻辑函数关联机制**-
  - componentSvgs 和 logicFunctions 使用相同的键名（如 'and-gate'）
  - 组件的 dataset.type 属性作为查找键，连接视觉和逻辑
  - 在信号传播时，通过 logicFunctions[targetType](inputStates) 动态调用对应的逻辑函数
  
## JS事件绑定和初始化
- ** DOM元素选择器的初始化**
  - 采用DOMContentLoaded 事件，确保DOM树完全构建后再执行JavaScript
  - querySelectorAll选择器便于后续批量绑定事件
- **侧边栏组件点击事件绑定**
  - 使用forEach方法为每个侧边栏组件项绑定独立的点击事件处理器
  - 通过 getAttribute('data-type') 获取组件类型，data-type 属性将HTML元素与JavaScript逻辑连接
  - 检查 componentSvgs[type] 是否存在，确保有对应的SVG模板
  - 动态创建div元素作为组件容器，即DOM创建
  - 将预定义的SVG模板插入到组件容器中
  - 设置组件的类型和唯一ID，用于后续的状态管理和事件处理
  - 调用统一的初始化函数 initializeComponent（）
